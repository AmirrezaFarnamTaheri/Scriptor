# Documentation screenshots

PNG assets in this folder are **committed to git** so README and docs render correctly on GitHub. They are generated by Playwright against the production Vite build with a mocked Tauri IPC bridge (`VITE_SCREENSHOT_MODE=true`).

| File | Captured surface | Primary use |
|------|------------------|-------------|
| `editor-preview.png` | Split editor + live Markdown preview | README hero, product overview |
| `workspace.png` | Source editor, vault tree, inspector rail | Guides, source-mode docs |
| `graph.png` | Knowledge graph dialog | Graph feature docs |
| `plugins.png` | Plugin marketplace | Plugin system docs |
| `command-palette.png` | Command palette | Navigation docs |
| `portal.png` | Portal quick capture | Portal docs |
| `knowledge-workbench.png` | Knowledge workbench | Workbench docs |
| `publish-center.png` | Publish center | Publishing docs |
| `settings.png` | Settings dialog | Settings / release docs |

## Regenerate

Uses system Edge/Chrome via Playwright `channel` (no bundled browser download):

```powershell
pnpm screenshots:capture:web
```

Full capture including desktop installer build:

```powershell
pnpm screenshots:capture
```

Override the browser channel:

```powershell
$env:PLAYWRIGHT_CHANNEL = 'chrome'
pnpm screenshots:capture:web
```

After UI changes that affect layout or copy, regenerate and commit the updated PNGs.
